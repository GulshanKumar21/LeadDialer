This executive report outlines the engineering deliverables, features, and platform upgrades successfully implemented for the ADYAPAN CRM system. All modules are fully operational, compile without errors, and have been successfully pushed to the central code repository.

1. Geotagged Attendance & Automated Payroll

  • Strict Geotagged Clock-In/Clock-Out: Implemented fine-grained GPS coordinate tracking during attendance submissions. Lateness triggers mandatory reason inputs for administrative reviews.
  • Grace Period Rules: Configured automated logic where clocking in between 11:05 AM and 11:10 AM is excused up to 3 times a month. Subsequent late entries automatically categorize as a Half Day.
  • Early Departure Guard: Programmed system verification requiring a minimum of 2.5 hours worked per session. Failing to meet this threshold automatically marks the attendance record as Absent.
  • Native PDF Payslip Generator: Built a native PdfDocument layout drawing engine directly on the device. Generates clean, download-ready payslips calculating worked days and deducting absent records. Integrates system sharing to send slips via mail/chat in one tap.
  • Tuesday Off Integration: Updated payroll calculation rules to exclude Tuesdays from mandatory working days, matching organizational policies.

2. Background Call Logs & Automated Dialer

  • Foreground Telephony Monitoring Service: Developed a persistent CallMonitorService utilizing a CallLog.Calls observer. Intercepts outgoing call statuses and computes the exact duration in seconds to prevent manual logging errors.
  • In-App Dialog Overlay: Designed a transparent dialog overlay rendering instantly upon call completion. Prompts counselors to choose feedback codes (Connected, Busy, Callback, etc.) with a fixed safety handler preventing crashes during late-night call feedback inputs.

3. Document Verification & Verification Portals

  • 4-Tab Onboarding Profile Form: Created a multi-page employee form categorizing fields under Personal, Job, Bank, and Documents (Aadhar, PAN, Marksheets) for profile submissions.
  • Secure Firebase Storage Uploads: Form documents upload to specific folders (employee_docs/{uid}/) and sync metadata into Firestore.
  • HR Documents Audit Panel: Built a 3-tab HR dashboard allowing executives to:
    • Distribute corporate guideline documents.
    • Monitor employee onboarding progress (e.g. 26/26 fields filled, 10/10 documents uploaded).
    • Inspect detail views of employee profiles and download submitted onboarding documents.

4. Cross-Platform Parity & SQLite Integration

  • Unified SQLite Storage (crm_database): Migrated the Flutter database to read/write directly from Room's crm_database file, ensuring data parity between native and Flutter interfaces on the same device.
  • Schema Upgrade: Aligned local schemas to match Room's Version 12, adding missing columns (duration in leads, note in call records).
  • Expected Sales & Admin Target: Integrated Firebase Realtime Database listeners. Employees can view targets and log daily Expected Sales directly from the dashboard, syncing instantly to management consoles.
  • Thought of the Day: Set up real-time listeners fetching daily motivation directly from Firestore settings, rendering it in a styled orange-accent box on the dashboard.

5. Security Controls & Access Roles

  • Supabase Integration: Integrated background synchronization (SupabaseSync) to fetch designations and roles from Supabase, updating Firestore profiles in real-time.
  • Role-Based Navigation: Automated menu styling to show or hide administrative actions (leaves approval, performance audits) depending on whether the user is logged in as Admin, HR, or Counselor.
